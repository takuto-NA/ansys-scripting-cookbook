# 🤝 コントリビューションガイド

`ansys-scripting-cookbook` への貢献を検討していただきありがとうございます！このプロジェクトは、エンジニア同士の知恵の共有で成り立っています。

## 1. 貢献の種類

- **レシピの追加**: 便利なスクリプトスニペットを教えてください。
- **バグ修正**: スクリプトのバグやドキュメントの誤字脱字の修正。
- **機能改善**: 既存スクリプトへの機能追加やリファクタリング。
- **ドキュメントの拡充**: 解説記事やトラブルシューティングの追加。

## 2. スクリプト投稿のルール

新しいスクリプトを追加する場合は、以下のガイドラインに従ってください。

- **動作確認**: 必ず実際の Ansys 環境で動作を確認してください。
- **バージョンの明記**: スクリプト冒頭に `# Tested on Ansys 2023 R2` のように動作確認バージョンを記載してください。
- **コメント**: 日本語（または英語）で分かりやすく処理内容を記述してください。
- **依存関係**: 原則として Ansys 標準搭載の IronPython 環境で動くように作成してください（外部ライブラリの追加インストールが必要なものは避ける）。

## 3. プルリクエスト (PR) の流れ

1. このリポジトリを Fork します。
2. 自分の環境でブランチを作成します (`git checkout -b feature/new-recipe`)。
3. 変更をコミットします (`git commit -am 'Add new mechanical snippet'`)。
4. オリジナルのリポジトリに PR を送信します。

## 4. 行動規範

お互いに敬意を払い、建設的なフィードバックを心がけましょう。
